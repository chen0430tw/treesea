MOROZ-Core 伪代码

Algorithm 1: MOROZ-Core

输入
	•	源层集合
\mathcal V=
\mathcal V_{\text{common-char}}
\cup
\mathcal V_{\text{common-pinyin}}
\cup
\mathcal V_{\text{whois/domain}}
\cup
\mathcal V_{\text{personal}}
\cup
\mathcal V_{\text{context}}
	•	最大长度 L
	•	候选模板集合 \mathcal T
	•	特征函数
\Phi=\{\phi_{\text{freq}},\phi_{\text{domain}},\phi_{\text{personal}},\phi_{\text{context}},\phi_{\text{syntax}}\}
	•	权重向量 w=(w_f,w_d,w_p,w_c,w_s)
	•	温度 \beta
	•	Gate 集合 \mathcal G
	•	预算 B
	•	Top-K 容量 K

输出
	•	Top-K 候选堆 TOP
	•	收缩统计 METRICS

⸻

Pseudocode

Algorithm MOROZ_CORE(V_layers, T_templates, L, Phi, w, beta, G, B, K):

1:  # ===== Stage 0: 初始化 =====
2:  V <- BUILD_SOURCE_POOL(V_layers)
3:  V_sku <- SKU_SELECT(V)                     # K-Warehouse: 先压缩底数
4:  PQ <- empty max-priority-queue            # 按 upper-bound / score 排序
5:  TOP <- empty min-heap of size K
6:  METRICS <- INIT_METRICS()

7:  # ===== Stage 1: 模板展开 =====
8:  for each template t in T_templates:
9:      p0 <- INIT_PREFIX(template=t)
10:     ub0 <- UPPER_BOUND(p0, Phi, w, beta)
11:     PUSH(PQ, (ub0, p0))

12: # ===== Stage 2: 主循环 =====
13: steps <- 0
14: while PQ not empty and steps < B:
15:     (ub, p) <- POP_MAX(PQ)

16:     # 2.1 Prefix Gate (ISSC: fail-fast)
17:     if not PREFIX_GATE_PASS(p, G):
18:         METRICS.reject_prefix += 1
19:         continue

20:     # 2.2 若前缀已完成，则进入完整评分
21:     if IS_COMPLETE(p, L):
22:         if not FULL_GATE_PASS(p, G):
23:             METRICS.reject_full += 1
24:             continue

25:         s_freq    <- phi_freq(p)
26:         s_domain  <- phi_domain(p)
27:         s_person  <- phi_personal(p)
28:         s_context <- phi_context(p)
29:         s_syntax  <- phi_syntax(p)

30:         score <- w_f*s_freq + w_d*s_domain + w_p*s_person
31:                  + w_c*s_context + w_s*s_syntax

32:         weight <- exp(beta * score)

33:         UPDATE_TOP_HEAP(TOP, (weight, p))
34:         UPDATE_METRICS(METRICS, p, score, weight)
35:         steps <- steps + 1
36:         continue

37:     # 2.3 否则做前缀扩展
38:     children <- EXPAND_PREFIX(p, V_sku)

39:     for each child in children:
40:         if not PREFIX_STRUCTURE_VALID(child):
41:             METRICS.reject_structure += 1
42:             continue

43:         ub_child <- UPPER_BOUND(child, Phi, w, beta)

44:         # K-Warehouse: 提前剪掉不可能进入头部的分支
45:         if HEAP_IS_FULL(TOP) and ub_child < CURRENT_THRESHOLD(TOP):
46:             METRICS.reject_bound += 1
47:             continue

48:         PUSH(PQ, (ub_child, child))
49:         METRICS.expanded += 1

50: # ===== Stage 3: 收缩统计 =====
51: METRICS.H   <- ENTROPY(TOP)
52: METRICS.C_q <- TOP_Q_COVERAGE(TOP, q)
53: METRICS.R   <- RETENTION_RATIO(METRICS)
54: METRICS.Theta_eff <- EFFECTIVE_THROUGHPUT(METRICS)

55: return TOP, METRICS

MOROZ-Core 子程序定义

BUILD_SOURCE_POOL

把五层源层合并成统一池，但保留层标签。

BUILD_SOURCE_POOL(V_layers):
    V <- []
    for each layer_name, layer_values in V_layers:
        for each token in layer_values:
            V.append( (token, layer_name) )
    return V


⸻

SKU_SELECT

K-Warehouse 的第一刀：先做词表精简。

SKU_SELECT(V):
    for each item in V:
        item.base_score <- SOURCE_PRIOR(item)
    return TOP_N_BY_SCORE(V)


⸻

UPPER_BOUND

给前缀一个理论上界，用于 best-first / 剪枝。

UPPER_BOUND(prefix, Phi, w, beta):
    partial <- PREFIX_SCORE(prefix)
    future  <- MAX_POSSIBLE_FUTURE_GAIN(prefix)
    return exp(beta * (partial + future))


⸻

PREFIX_GATE_PASS / FULL_GATE_PASS

ISSC 的关键：前缀能判的先判，完整的再判。

PREFIX_GATE_PASS(prefix, G):
    for each gate in G.prefix_gates:
        if gate(prefix) == False:
            return False
    return True

FULL_GATE_PASS(candidate, G):
    for each gate in G.full_gates:
        if gate(candidate) == False:
            return False
    return True


⸻

EXPAND_PREFIX

按模板与源层规则扩展下一步。

EXPAND_PREFIX(prefix, V_sku):
    next_slots <- NEXT_ALLOWED_SLOTS(prefix)
    children <- []
    for each slot_rule in next_slots:
        for each token in FILTER_BY_SLOT(V_sku, slot_rule):
            children.append(prefix + token)
    return children

⸻

Algorithm 2: MOROZ-HCE

输入
	•	TASK_SPEC：任务定义
	•	CORE_CONFIG：MOROZ-Core 配置
	•	SHARD_CONFIG：分片配置
	•	RUNTIME_CONFIG：运行时配置

输出
	•	全局结果 GLOBAL_TOP
	•	全局运行资产目录

⸻

Pseudocode

Algorithm MOROZ_HCE(TASK_SPEC, CORE_CONFIG, SHARD_CONFIG, RUNTIME_CONFIG):

1:  # ===== Stage 0: 初始化运行目录 =====
2:  RUN_ID <- CREATE_RUN_ID()
3:  MAKE_DIRS(RUN_ID, ["logs", "results", "checkpoints", "metrics", "artifacts"])

4:  # ===== Stage 1: 候选空间切分 =====
5:  SHARDS <- BUILD_SHARDS(TASK_SPEC, SHARD_CONFIG)

6:  # ===== Stage 2: 调度器启动 =====
7:  QUEUE <- INIT_QUEUE(SHARDS)
8:  GLOBAL_TOP <- empty global heap
9:  GLOBAL_METRICS <- INIT_GLOBAL_METRICS()

10: while QUEUE not empty:
11:     shard <- DISPATCH_NEXT_SHARD(QUEUE, RUNTIME_CONFIG)

12:     # 2.1 恢复检查点
13:     state <- TRY_RESTORE_CHECKPOINT(RUN_ID, shard)

14:     # 2.2 执行 MOROZ-Core
15:     local_top, local_metrics <- MOROZ_CORE_FROM_STATE(
16:         shard, CORE_CONFIG, state
17:     )

18:     # 2.3 落盘与归档
19:     SAVE_LOCAL_RESULTS(RUN_ID, shard, local_top, local_metrics)
20:     SAVE_CHECKPOINT(RUN_ID, shard, local_top, local_metrics)

21:     # 2.4 汇总
22:     MERGE_GLOBAL_TOP(GLOBAL_TOP, local_top)
23:     MERGE_GLOBAL_METRICS(GLOBAL_METRICS, local_metrics)

24:     # 2.5 故障处理
25:     if SHARD_FAILED(shard):
26:         if RETRY_ALLOWED(shard):
27:             REQUEUE(shard)
28:         else:
29:             MARK_FAILED(RUN_ID, shard)

30: # ===== Stage 3: 最终汇总 =====
31: FINALIZE_RUN_SUMMARY(RUN_ID, GLOBAL_TOP, GLOBAL_METRICS)
32: EXPORT_REPORT(RUN_ID)

33: return GLOBAL_TOP, GLOBAL_METRICS

from __future__ import annotations

from dataclasses import dataclass
from typing import Callable, Iterable, Sequence

from .types import Candidate, FeatureWeights, LayerName, ScoreBreakdown, SourceToken


FeatureFn = Callable[[Candidate], float]


@dataclass
class MSCMConfig:
    beta: float = 1.0
    source_top_n: int | None = None


@dataclass
class MSCM:
    common_char: Sequence[SourceToken]
    common_pinyin: Sequence[SourceToken]
    whois_domain: Sequence[SourceToken]
    personal: Sequence[SourceToken]
    context: Sequence[SourceToken]
    weights: FeatureWeights
    phi_freq: FeatureFn
    phi_domain: FeatureFn
    phi_personal: FeatureFn
    phi_context: FeatureFn
    phi_syntax: FeatureFn
    config: MSCMConfig = MSCMConfig()

    def build_source_pool(self) -> list[SourceToken]:
        pool = (
            list(self.common_char)
            + list(self.common_pinyin)
            + list(self.whois_domain)
            + list(self.personal)
            + list(self.context)
        )
        pool.sort(key=lambda x: x.prior, reverse=True)
        if self.config.source_top_n is not None:
            pool = pool[: self.config.source_top_n]
        return pool

    def score(self, candidate: Candidate) -> ScoreBreakdown:
        f = self.phi_freq(candidate)
        d = self.phi_domain(candidate)
        p = self.phi_personal(candidate)
        c = self.phi_context(candidate)
        s = self.phi_syntax(candidate)

        total = (
            self.weights.freq * f
            + self.weights.domain * d
            + self.weights.personal * p
            + self.weights.context * c
            + self.weights.syntax * s
        )
        return ScoreBreakdown(
            freq=f,
            domain=d,
            personal=p,
            context=c,
            syntax=s,
            total=total,
        )

    def weight(self, candidate: Candidate) -> float:
        import math
        return math.exp(self.config.beta * self.score(candidate).total)

三、k_warehouse.py

from __future__ import annotations

import heapq
from dataclasses import dataclass
from typing import Callable, Iterable, Sequence

from .types import Candidate, SearchMetrics, SourceToken


PrefixGate = Callable[[Candidate], bool]
FullGate = Callable[[Candidate], bool]
UpperBoundFn = Callable[[Candidate], float]
StructureFn = Callable[[Candidate], bool]
ScoreFn = Callable[[Candidate], float]


@dataclass
class KWConfig:
    max_len: int
    budget: int = 10000
    top_k: int = 50


@dataclass
class KWResult:
    top: list[tuple[float, Candidate]]
    metrics: SearchMetrics


class KWarehouse:
    def __init__(
        self,
        source_pool: Sequence[SourceToken],
        score_fn: ScoreFn,
        upper_bound_fn: UpperBoundFn,
        prefix_gate: PrefixGate,
        full_gate: FullGate,
        structure_valid: StructureFn,
        config: KWConfig,
    ) -> None:
        self.source_pool = list(source_pool)
        self.score_fn = score_fn
        self.upper_bound_fn = upper_bound_fn
        self.prefix_gate = prefix_gate
        self.full_gate = full_gate
        self.structure_valid = structure_valid
        self.config = config

    def run(self, seeds: Iterable[Candidate] | None = None) -> KWResult:
        pq: list[tuple[float, int, Candidate]] = []
        top: list[tuple[float, Candidate]] = []
        metrics = SearchMetrics()
        counter = 0

        if seeds is None:
            seeds = [Candidate(tokens=())]

        for seed in seeds:
            ub = self.upper_bound_fn(seed)
            heapq.heappush(pq, (-ub, counter, seed))
            counter += 1

        while pq and metrics.accepted < self.config.budget:
            _, _, cand = heapq.heappop(pq)

            if not self.prefix_gate(cand):
                metrics.reject_prefix += 1
                continue

            if len(cand.tokens) >= self.config.max_len:
                if not self.full_gate(cand):
                    metrics.reject_full += 1
                    continue
                score = self.score_fn(cand)
                self._push_top(top, score, cand)
                metrics.accepted += 1
                continue

            for token in self.source_pool:
                child = cand.extend(token)
                if not self.structure_valid(child):
                    metrics.reject_structure += 1
                    continue

                ub = self.upper_bound_fn(child)
                if len(top) >= self.config.top_k and ub < top[0][0]:
                    metrics.reject_bound += 1
                    continue

                heapq.heappush(pq, (-ub, counter, child))
                counter += 1
                metrics.expanded += 1

        top.sort(key=lambda x: x[0], reverse=True)
        return KWResult(top=top, metrics=metrics)

    def _push_top(self, top: list[tuple[float, Candidate]], score: float, cand: Candidate) -> None:
        if len(top) < self.config.top_k:
            heapq.heappush(top, (score, cand))
        else:
            if score > top[0][0]:
                heapq.heapreplace(top, (score, cand))

四、issc.py

from __future__ import annotations

import math
from dataclasses import dataclass
from typing import Iterable, Sequence

from .types import Candidate, SearchMetrics


@dataclass
class ISSCStats:
    entropy: float
    top_q_coverage: float
    retention_ratio: float
    theta_eff: float


@dataclass
class ISSCResult:
    ranked: list[tuple[float, Candidate]]
    stats: ISSCStats


class ISSC:
    def __init__(self, q: int = 5) -> None:
        self.q = q

    def collapse(
        self,
        ranked: Sequence[tuple[float, Candidate]],
        metrics: SearchMetrics,
        elapsed_seconds: float,
    ) -> ISSCResult:
        if not ranked:
            return ISSCResult(
                ranked=[],
                stats=ISSCStats(
                    entropy=0.0,
                    top_q_coverage=0.0,
                    retention_ratio=0.0,
                    theta_eff=0.0,
                ),
            )

        weights = [max(score, 0.0) for score, _ in ranked]
        s = sum(weights)
        probs = [w / s for w in weights] if s > 0 else [0.0 for _ in weights]

        entropy = -sum(p * math.log(p) for p in probs if p > 0)
        top_q = sum(probs[: min(self.q, len(probs))])

        total_seen = (
            metrics.expanded
            + metrics.reject_prefix
            + metrics.reject_full
            + metrics.reject_structure
            + metrics.reject_bound
            + metrics.accepted
        )
        retention_ratio = (metrics.accepted / total_seen) if total_seen > 0 else 0.0
        theta_eff = (metrics.accepted / elapsed_seconds) if elapsed_seconds > 0 else 0.0

        return ISSCResult(
            ranked=list(ranked),
            stats=ISSCStats(
                entropy=entropy,
                top_q_coverage=top_q,
                retention_ratio=retention_ratio,
                theta_eff=theta_eff,
            ),
        )

from __future__ import annotations

import time
from dataclasses import dataclass
from typing import Callable, Iterable, Sequence

from .issc import ISSC, ISSCResult
from .k_warehouse import KWConfig, KWResult, KWarehouse
from .mscm import MSCM
from .types import Candidate, SearchMetrics


PrefixGate = Callable[[Candidate], bool]
FullGate = Callable[[Candidate], bool]
StructureFn = Callable[[Candidate], bool]
SeedBuilder = Callable[[], Iterable[Candidate]]


@dataclass
class MOROZCoreConfig:
    max_len: int
    budget: int = 10000
    top_k: int = 50
    collapse_q: int = 5


@dataclass
class MOROZCoreResult:
    kw_result: KWResult
    issc_result: ISSCResult
    elapsed_seconds: float


class MOROZCore:
    """
    MOROZ-Core = MSCM -> K-Warehouse -> ISSC

    责任边界：
    - MSCM: 多源候选建模与评分
    - K-Warehouse: 候选压缩、搜索、Top-K
    - ISSC: 收缩统计与原地坍缩观测
    """

    def __init__(
        self,
        mscm: MSCM,
        config: MOROZCoreConfig,
        prefix_gate: PrefixGate,
        full_gate: FullGate,
        structure_valid: StructureFn,
        seed_builder: SeedBuilder | None = None,
    ) -> None:
        self.mscm = mscm
        self.config = config
        self.prefix_gate = prefix_gate
        self.full_gate = full_gate
        self.structure_valid = structure_valid
        self.seed_builder = seed_builder or self._default_seed_builder

    # =========================
    # Public API
    # =========================
    def run(self) -> MOROZCoreResult:
        """
        运行完整 MOROZ-Core 闭环：
        1) MSCM 构建 source pool
        2) K-Warehouse 做压缩搜索
        3) ISSC 做收缩统计
        """
        t0 = time.perf_counter()

        source_pool = self.mscm.build_source_pool()

        kw = KWarehouse(
            source_pool=source_pool,
            score_fn=self._score_fn,
            upper_bound_fn=self._upper_bound_fn,
            prefix_gate=self.prefix_gate,
            full_gate=self.full_gate,
            structure_valid=self.structure_valid,
            config=KWConfig(
                max_len=self.config.max_len,
                budget=self.config.budget,
                top_k=self.config.top_k,
            ),
        )

        seeds = list(self.seed_builder())
        kw_result = kw.run(seeds=seeds)

        t1 = time.perf_counter()

        issc = ISSC(q=self.config.collapse_q)
        issc_result = issc.collapse(
            ranked=kw_result.top,
            metrics=kw_result.metrics,
            elapsed_seconds=t1 - t0,
        )

        return MOROZCoreResult(
            kw_result=kw_result,
            issc_result=issc_result,
            elapsed_seconds=t1 - t0,
        )

    def explain_candidate(self, candidate: Candidate) -> dict:
        """
        给单个候选输出 MSCM 五层评分拆解。
        方便你之后做 debug / report / 可视化。
        """
        breakdown = self.mscm.score(candidate)
        return {
            "text": candidate.text,
            "freq": breakdown.freq,
            "domain": breakdown.domain,
            "personal": breakdown.personal,
            "context": breakdown.context,
            "syntax": breakdown.syntax,
            "total": breakdown.total,
            "weight": self.mscm.weight(candidate),
        }

    # =========================
    # Internal scoring
    # =========================
    def _score_fn(self, candidate: Candidate) -> float:
        return self.mscm.score(candidate).total

    def _upper_bound_fn(self, candidate: Candidate) -> float:
        """
        一个安全保守的上界函数：
        当前得分 + 剩余长度 * 每步理论最大增益

        这里先用简单版，后面你可以换成更精确的 per-layer upper bound。
        """
        current = self._score_fn(candidate)
        remain = max(0, self.config.max_len - len(candidate.tokens))
        max_gain_per_step = 1.0
        return current + remain * max_gain_per_step

    def _default_seed_builder(self) -> Iterable[Candidate]:
        yield Candidate(tokens=())

from __future__ import annotations

import json
import math
from dataclasses import asdict, dataclass
from pathlib import Path
from typing import Sequence

from .types import Candidate, SearchMetrics


@dataclass
class CollapseStats:
    entropy: float
    top_q_coverage: float
    retention_ratio: float
    theta_eff: float
    accepted: int
    expanded: int
    total_seen: int


def entropy_from_scores(scores: Sequence[float]) -> float:
    if not scores:
        return 0.0
    positive = [max(s, 0.0) for s in scores]
    s = sum(positive)
    if s <= 0:
        return 0.0
    probs = [x / s for x in positive]
    return -sum(p * math.log(p) for p in probs if p > 0)


def top_q_coverage_from_scores(scores: Sequence[float], q: int) -> float:
    if not scores:
        return 0.0
    positive = [max(s, 0.0) for s in scores]
    s = sum(positive)
    if s <= 0:
        return 0.0
    probs = [x / s for x in positive]
    return sum(probs[: min(q, len(probs))])


def retention_ratio(metrics: SearchMetrics) -> float:
    total_seen = (
        metrics.expanded
        + metrics.reject_prefix
        + metrics.reject_full
        + metrics.reject_structure
        + metrics.reject_bound
        + metrics.accepted
    )
    if total_seen <= 0:
        return 0.0
    return metrics.accepted / total_seen


def effective_throughput(metrics: SearchMetrics, elapsed_seconds: float) -> float:
    if elapsed_seconds <= 0:
        return 0.0
    return metrics.accepted / elapsed_seconds


def build_collapse_stats(
    ranked: Sequence[tuple[float, Candidate]],
    metrics: SearchMetrics,
    elapsed_seconds: float,
    q: int = 5,
) -> CollapseStats:
    scores = [score for score, _ in ranked]
    total_seen = (
        metrics.expanded
        + metrics.reject_prefix
        + metrics.reject_full
        + metrics.reject_structure
        + metrics.reject_bound
        + metrics.accepted
    )
    return CollapseStats(
        entropy=entropy_from_scores(scores),
        top_q_coverage=top_q_coverage_from_scores(scores, q=q),
        retention_ratio=retention_ratio(metrics),
        theta_eff=effective_throughput(metrics, elapsed_seconds),
        accepted=metrics.accepted,
        expanded=metrics.expanded,
        total_seen=total_seen,
    )


def save_metrics_json(path: str | Path, stats: CollapseStats) -> None:
    path = Path(path)
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_text(json.dumps(asdict(stats), indent=2, ensure_ascii=False), encoding="utf-8")

from __future__ import annotations

import json
import time
import uuid
from dataclasses import asdict, dataclass, field
from pathlib import Path
from typing import Callable, Iterable, Sequence

from moroz.core.metrics import save_metrics_json
from moroz.core.moroz_core import MOROZCore, MOROZCoreResult
from moroz.core.types import Candidate


ShardBuilder = Callable[[], list[dict]]
CoreFactory = Callable[[dict], MOROZCore]


@dataclass
class HCEConfig:
    run_root: str = "runs"
    keep_top_k_global: int = 50
    save_checkpoints: bool = True
    save_local_ranked: bool = True


@dataclass
class ShardResult:
    shard_id: str
    top: list[tuple[float, Candidate]]
    elapsed_seconds: float
    stats: dict


@dataclass
class HCESummary:
    run_id: str
    shard_count: int
    total_elapsed_seconds: float
    global_top_count: int
    failed_shards: list[str] = field(default_factory=list)


class MOROZHCE:
    """
    MOROZ-HCE: HCE 运行时总装层
    负责：
    - run 目录管理
    - shard 调度
    - 本地结果保存
    - checkpoint 保存
    - 全局 top 汇总
    """

    def __init__(
        self,
        shard_builder: ShardBuilder,
        core_factory: CoreFactory,
        config: HCEConfig | None = None,
    ) -> None:
        self.shard_builder = shard_builder
        self.core_factory = core_factory
        self.config = config or HCEConfig()

    def run(self) -> tuple[list[tuple[float, str]], HCESummary]:
        run_id = self._create_run_id()
        run_dir = self._prepare_run_dirs(run_id)

        shards = self.shard_builder()
        global_top: list[tuple[float, str]] = []
        failed_shards: list[str] = []

        t0 = time.perf_counter()

        for shard in shards:
            shard_id = shard["shard_id"]
            try:
                core = self.core_factory(shard)
                result = core.run()
                self._save_shard_result(run_dir, shard_id, result, shard)
                self._merge_global_top(global_top, result)
                if self.config.save_checkpoints:
                    self._save_checkpoint(run_dir, shard_id, result, shard)
            except Exception as e:
                failed_shards.append(shard_id)
                self._save_error(run_dir, shard_id, e)

        t1 = time.perf_counter()

        summary = HCESummary(
            run_id=run_id,
            shard_count=len(shards),
            total_elapsed_seconds=t1 - t0,
            global_top_count=len(global_top),
            failed_shards=failed_shards,
        )
        self._save_summary(run_dir, summary, global_top)
        return global_top, summary

    # =========================
    # Internal helpers
    # =========================
    def _create_run_id(self) -> str:
        ts = time.strftime("%Y%m%d_%H%M%S")
        suffix = uuid.uuid4().hex[:8]
        return f"moroz_{ts}_{suffix}"

    def _prepare_run_dirs(self, run_id: str) -> Path:
        run_dir = Path(self.config.run_root) / run_id
        for sub in ["logs", "results", "checkpoints", "metrics", "artifacts"]:
            (run_dir / sub).mkdir(parents=True, exist_ok=True)
        return run_dir

    def _save_shard_result(
        self,
        run_dir: Path,
        shard_id: str,
        result: MOROZCoreResult,
        shard_spec: dict,
    ) -> None:
        result_path = run_dir / "results" / f"{shard_id}.json"
        metrics_path = run_dir / "metrics" / f"{shard_id}_metrics.json"

        payload = {
            "shard_id": shard_id,
            "shard_spec": shard_spec,
            "elapsed_seconds": result.elapsed_seconds,
            "ranked": [
                {"score": float(score), "text": cand.text}
                for score, cand in result.issc_result.ranked
            ],
        }
        result_path.write_text(
            json.dumps(payload, indent=2, ensure_ascii=False),
            encoding="utf-8",
        )

        save_metrics_json(metrics_path, result.issc_result.stats)

    def _save_checkpoint(
        self,
        run_dir: Path,
        shard_id: str,
        result: MOROZCoreResult,
        shard_spec: dict,
    ) -> None:
        ckpt_path = run_dir / "checkpoints" / f"{shard_id}.json"
        payload = {
            "shard_id": shard_id,
            "shard_spec": shard_spec,
            "elapsed_seconds": result.elapsed_seconds,
            "top": [
                {"score": float(score), "text": cand.text}
                for score, cand in result.issc_result.ranked
            ],
            "stats": asdict(result.issc_result.stats),
        }
        ckpt_path.write_text(
            json.dumps(payload, indent=2, ensure_ascii=False),
            encoding="utf-8",
        )

    def _save_error(self, run_dir: Path, shard_id: str, e: Exception) -> None:
        err_path = run_dir / "logs" / f"{shard_id}.err.txt"
        err_path.write_text(str(e), encoding="utf-8")

    def _merge_global_top(
        self,
        global_top: list[tuple[float, str]],
        result: MOROZCoreResult,
    ) -> None:
        for score, cand in result.issc_result.ranked:
            global_top.append((float(score), cand.text))

        global_top.sort(key=lambda x: x[0], reverse=True)
        del global_top[self.config.keep_top_k_global :]

    def _save_summary(
        self,
        run_dir: Path,
        summary: HCESummary,
        global_top: list[tuple[float, str]],
    ) -> None:
        summary_path = run_dir / "summary.json"
        payload = {
            "summary": asdict(summary),
            "global_top": [
                {"score": score, "text": text}
                for score, text in global_top
            ],
        }
        summary_path.write_text(
            json.dumps(payload, indent=2, ensure_ascii=False),
            encoding="utf-8",
        )

from moroz.core.types import Candidate, FeatureWeights, SourceToken
from moroz.hce import HCEConfig, MOROZHCE


def contains_any(text: str, keys: list[str]) -> float:
    return 1.0 if any(k in text for k in keys) else 0.0


def make_mscm() -> MSCM:
    common_pinyin = [
        SourceToken("cat", "common_pinyin", 0.90),
        SourceToken("kitty", "common_pinyin", 0.85),
        SourceToken("kitten", "common_pinyin", 0.80),
        SourceToken("lazy", "common_pinyin", 0.60),
        SourceToken("sleepy", "common_pinyin", 0.70),
    ]

    whois_domain = [
        SourceToken("mail", "whois_domain", 0.50),
        SourceToken("cloud", "whois_domain", 0.55),
    ]

    personal = [
        SourceToken("mimi", "personal", 0.95),
    ]

    context = [
        SourceToken("photo", "context", 0.80),
    ]

    common_char = []

    def phi_freq(c: Candidate) -> float:
        if not c.tokens:
            return 0.0
        return sum(t.prior for t in c.tokens) / len(c.tokens)

    def phi_domain(c: Candidate) -> float:
        return contains_any(c.text, ["mail", "cloud"])

    def phi_personal(c: Candidate) -> float:
        return contains_any(c.text, ["mimi"])

    def phi_context(c: Candidate) -> float:
        return contains_any(c.text, ["photo", "cat", "kitty"])

    def phi_syntax(c: Candidate) -> float:
        return 1.0 if len(c.tokens) <= 3 else 0.0

    return MSCM(
        common_char=common_char,
        common_pinyin=common_pinyin,
        whois_domain=whois_domain,
        personal=personal,
        context=context,
        weights=FeatureWeights(
            freq=1.0,
            domain=0.7,
            personal=0.8,
            context=1.0,
            syntax=1.0,
        ),
        phi_freq=phi_freq,
        phi_domain=phi_domain,
        phi_personal=phi_personal,
        phi_context=phi_context,
        phi_syntax=phi_syntax,
    )


def prefix_gate(c: Candidate) -> bool:
    return len(c.tokens) <= 3


def full_gate(c: Candidate) -> bool:
    return len(c.tokens) == 3


def structure_valid(c: Candidate) -> bool:
    return True


def build_shards() -> list[dict]:
    return [
        {"shard_id": "shard_000", "tag": "alpha"},
        {"shard_id": "shard_001", "tag": "beta"},
        {"shard_id": "shard_002", "tag": "gamma"},
    ]


def core_factory(shard_spec: dict) -> MOROZCore:
    mscm = make_mscm()
    return MOROZCore(
        mscm=mscm,
        config=MOROZCoreConfig(
            max_len=3,
            budget=200,
            top_k=10,
            collapse_q=5,
        ),
        prefix_gate=prefix_gate,
        full_gate=full_gate,
        structure_valid=structure_valid,
    )


runner = MOROZHCE(
    shard_builder=build_shards,
    core_factory=core_factory,
    config=HCEConfig(
        run_root="runs",
        keep_top_k_global=20,
        save_checkpoints=True,
        save_local_ranked=True,
    ),
)

global_top, summary = runner.run()

print("=== GLOBAL TOP ===")
for score, text in global_top:
    print(f"{score:.4f}  {text}")

print("\n=== SUMMARY ===")
print(summary)

为什么不该只做成 hashcat 那种纯命令行感

Hashcat 那种交互很适合：
	•	单任务
	•	明确模式
	•	直接开跑
	•	看速度/状态/恢复

但 MOROZ 不只是“执行器”，它还有：
	•	MSCM 多源候选层
	•	K-Warehouse 压缩层
	•	ISSC 收缩层
	•	HCE 分片/恢复/结果层

这意味着你会经常想看这些东西：
	•	当前用的是哪一层 source
	•	哪些 gate 在工作
	•	哪些 shard 完成了
	•	Top-K 在怎么变
	•	熵 H、覆盖率 C_q、保留率 R 在怎么走
	•	哪个 run 的 summary 最好

这些内容如果全靠 CLI 参数和 stdout 刷屏，会很快很痛苦。

MOROZ 本体不该像纯 hashcat，而应该是“CLI 可脚本化 + TUI 可观测”的双模系统。

也就是：

	•	盯系统时：TUI

⸻

1. Run Summary 页

这是默认首页。

作用

给你一眼看出：

必备信息

	•	当前 source 层启用情况

2. Collapse Monitor 页

这是 MOROZ 最有灵魂的一页。

作用

专门看“收缩有没有发生”。

内容

│ Collapse Monitor                                     │
├──────────────────────────────────────────────────────┤
│ Entropy H                    1.823                   │

六、它和 hashcat 风格的区别

hashcat 风格

更像：
	•	一个任务在跑
	•	我看速度、温度、ETA

MOROZ-TUI

更像：
	•	一个系统在运转
	•	我看：
	•	候选是怎么来的
	•	是怎么被压缩的
	•	哪些 shard 在活
	•	收缩有没有发生
	•	结果是不是越来越集中

所以 MOROZ 的 TUI 应该更像：

任务编排控制台
而不是
单任务撞库面板

from __future__ import annotations


def handle_command(command: str) -> str:
    cmd = command.strip().lower()

    if not cmd:
        return ""

    if cmd == "pause":
        return "Requested: pause"
    if cmd == "resume":
        return "Requested: resume"
    if cmd.startswith("retry"):
        return f"Requested: {command}"
    if cmd.startswith("explain "):
        target = command[len("explain "):].strip()
        return f"Explain candidate: {target}"
    if cmd == "export summary":
        return "Requested: export summary"
    if cmd == "help":
        return "Commands: pause, resume, retry <shard>, explain <candidate>, export summary"

    return f"Unknown command: {command}"

from __future__ import annotations

from textual.app import App, ComposeResult
from textual.binding import Binding
from textual.containers import Container, Horizontal, Vertical
from textual.widgets import Header

from .actions import handle_command
from .state import (
    CandidateRow,
    CollapseState,
    MOROZTuiState,
    ShardRow,
    SummaryState,
)
from .widgets.candidates_panel import CandidatesPanel
from .widgets.collapse_panel import CollapsePanel
from .widgets.command_bar import CommandBar
from .widgets.footer_bar import FooterBar
from .widgets.shards_panel import ShardsPanel
from .widgets.summary_panel import SummaryPanel


class MOROZTui(App):
    CSS = """
    Screen {
        layout: vertical;
    }

    #top_row {
        height: 16;
    }

    #mid_row {
        height: 1fr;
    }

    #summary_panel, #collapse_panel, #candidates_panel, #shards_panel {
        border: round $accent;
        padding: 1;
    }

    #command_bar {
        height: 3;
        border: round $accent;
    }

    #footer_bar {
        height: 1;
    }
    """

    BINDINGS = [
        Binding("q", "quit", "Quit"),
        Binding("f2", "focus_summary", "Summary"),
        Binding("f4", "focus_shards", "Shards"),
        Binding("f5", "focus_candidates", "Candidates"),
        Binding("f6", "focus_collapse", "Collapse"),
        Binding("r", "retry_selected", "Retry"),
        Binding("p", "pause_run", "Pause"),
        Binding("s", "save_snapshot", "Save"),
    ]

    def __init__(self) -> None:
        super().__init__()
        self.state = MOROZTuiState(
            summary=SummaryState(
                run_id="moroz_demo_001",
                mode="HCE",
                state="RUNNING",
                elapsed="00:12:41",
                active_shards=2,
                completed_shards=8,
                failed_shards=1,
                requeued_shards=1,
                global_top_k=10,
                budget_used="38%",
                source_layers=5,
            ),
            collapse=CollapseState(
                entropy=1.8234,
                top_q_coverage=0.7412,
                retention_ratio=0.0631,
                theta_eff=421.7,
                bound_rejects=18231,
                prefix_rejects=9132,
                structure_rejects=2011,
            ),
            candidates=[
                CandidateRow(1, 8.4210, "mimicatphoto", "personal+context"),
                CandidateRow(2, 8.1050, "kittycloudmimi", "domain+personal"),
                CandidateRow(3, 7.9920, "catmailphoto", "domain+context"),
                CandidateRow(4, 7.8110, "mimisleepycat", "personal+freq"),
            ],
            shards=[
                ShardRow("shard_000", "DONE", "00:00:13", 50, 1820, "ok"),
                ShardRow("shard_001", "RUN", "00:00:08", 14, 433, "active"),
                ShardRow("shard_002", "FAIL", "00:00:03", 0, 0, "retry=1"),
                ShardRow("shard_003", "QUEUE", "--", 0, 0, "waiting"),
            ],
        )

    def compose(self) -> ComposeResult:
        yield Header(show_clock=True)

        with Horizontal(id="top_row"):
            self.summary_panel = SummaryPanel(id="summary_panel")
            self.collapse_panel = CollapsePanel(id="collapse_panel")
            yield self.summary_panel
            yield self.collapse_panel

        with Horizontal(id="mid_row"):
            self.candidates_panel = CandidatesPanel(id="candidates_panel")
            self.shards_panel = ShardsPanel(id="shards_panel")
            yield self.candidates_panel
            yield self.shards_panel

        self.command_bar = CommandBar(id="command_bar")
        yield self.command_bar

        self.footer_bar = FooterBar(id="footer_bar")
        yield self.footer_bar

    def on_mount(self) -> None:
        self.refresh_all()

    def refresh_all(self) -> None:
        self.summary_panel.update(self.summary_panel.render_summary(self.state.summary))
        self.collapse_panel.update(self.collapse_panel.render_collapse(self.state.collapse))
        self.candidates_panel.update_rows(self.state.candidates)
        self.shards_panel.update_rows(self.state.shards)

    def action_focus_summary(self) -> None:
        self.summary_panel.focus()

    def action_focus_shards(self) -> None:
        self.shards_panel.focus()

    def action_focus_candidates(self) -> None:
        self.candidates_panel.focus()

    def action_focus_collapse(self) -> None:
        self.collapse_panel.focus()

    def action_retry_selected(self) -> None:
        self.state.command_output = "Retry requested"
        self.footer_bar.update(f"{FooterBar.DEFAULT_TEXT} | Retry requested")

    def action_pause_run(self) -> None:
        self.state.command_output = "Pause requested"
        self.footer_bar.update(f"{FooterBar.DEFAULT_TEXT} | Pause requested")

    def action_save_snapshot(self) -> None:
        self.state.command_output = "Snapshot saved"
        self.footer_bar.update(f"{FooterBar.DEFAULT_TEXT} | Snapshot saved")

    def on_input_submitted(self, event) -> None:
        if event.input.id != "command_input":
            return
        result = handle_command(event.value)
        self.footer_bar.update(f"{FooterBar.DEFAULT_TEXT} | {result}")
        event.input.value = ""


if __name__ == "__main__":
    MOROZTui().run()

一、先定义 QCU 接口

建议新建：

moroz/core/qcu.py

from __future__ import annotations

from dataclasses import dataclass
from .types import Candidate


@dataclass
class QCUScore:
    collapse_gain: float
    coherence: float
    stability: float
    total: float


class QCU:
    """
    QCU: 坍缩增强单元
    先作为可插拔重排器存在，不直接污染 MOROZ-Core 本体。
    """

    def score(self, candidate: Candidate) -> QCUScore:
        text = candidate.text

        # toy 版：你后面可以替换成真实 QCU 逻辑
        coherence = 1.0 if any(x in text for x in ["cat", "kitty", "kitten"]) else 0.0
        stability = 1.0 if any(x in text for x in ["mimi", "photo", "cloud", "mail"]) else 0.0
        collapse_gain = 0.5 * coherence + 0.5 * stability

        total = collapse_gain + coherence + stability
        return QCUScore(
            collapse_gain=collapse_gain,
            coherence=coherence,
            stability=stability,
            total=total,
        )

def _score_fn(self, candidate: Candidate) -> float:
    base = self.mscm.score(candidate).total
    if self.qcu is None or self.qcu_weight <= 0.0:
        return base
    qcu_score = self.qcu.score(candidate).total
    return base + self.qcu_weight * qcu_score

但底层真正发生的是：

候选空间先被压扁，再被 QCU 快速选中。

⸻

一、QCU 秒的不是“密码学强度”，而是“剩余不确定性”

哈希本身不会因为 QCU 就突然失效。

QCU 真正擅长的是：
	•	对候选集做相位化
	•	对高一致性候选做增益
	•	对低一致性候选做快速压低
	•	让系统尽快掉进一个 attractor

所以如果你前面已经把空间压成：
	•	100 个候选
	•	1000 个候选
	•	或者一个很强模板家族

那 QCU 看起来就会非常快，甚至“秒”。

但如果你给它的是：
	•	高熵随机密码
	•	大空间强哈希
	•	没有任何上下文

那它不会神奇地把密码学打爆。

⸻

二、为什么在你的体系里会显得特别快

因为你的体系不是单独的 QCU，而是通常隐含了这些东西：

1. 前端已经做了熵减

比如：
	•	K-Warehouse
	•	ISSC
	•	MSCM
	•	whois / 常用汉字 / 个人线索
	•	模板约束
	•	弱口令结构

这些东西会先把空间从“宇宙级”压到“可坍缩级”。

QCU 接手的时候，面对的不是原始问题，而是一个已经被削成薄片的问题。

⸻

2. QCU 是局部坍缩器，不是全局枚举器

它不像传统暴力那样：
	•	一个一个试
	•	全空间线性扫

它更像：
	•	先把候选态并存
	•	再通过动力学把某些态快速放大
	•	最后读出

所以在小而窄的搜索区里，它会有一种很强的“秒感”。

⸻

3. 弱口令问题本来就很容易出现相变

弱口令往往具有这些特点：
	•	token 重复
	•	模板固定
	•	个人词明显
	•	年份/域名/昵称混搭
	•	熵很低

这种时候，系统不是在“解一个真正随机密码”，而是在“识别一个高模式化对象”。

而模式化对象，最适合被 QCU 这种坍缩式系统处理。

⸻

三、为什么这次猫咪 benchmark 没有表现成“秒杀一切”

因为这次 benchmark 故意换了问题类型。

猫咪测试测的是：
	•	候选排序能力
	•	attractor 偏好
	•	串联系统结构
	•	frontier 压缩后的行为

不是在测：
	•	某个真实弱 hash 的命中效率

所以它问的是：

QCU 喜欢什么候选结构？

不是：

QCU 能不能把一个被强约束的弱口令瞬间打中？

这两个实验目标不同。

⸻

四、你可以把它想成两个层级

层级 A：通用排序实验

问题是：
	•	谁把候选排得更前
	•	谁更不容易跑偏

这里 QCU 不一定占优，因为它不是为“全局排序”设计的。

⸻

层级 B：窄空间坍缩实验

问题是：
	•	候选已经被压缩到很窄
	•	现在谁能最快锁定正确那一簇

这里 QCU 就会很有杀伤力。

所以“秒解 hash”的感觉，多半来自 层级 B。

⸻

五、一句话解释你感受到的差异

你之前看到的 QCU 很强，是因为它面对的是：

低熵、强模板、强约束、窄候选空间的问题。

而不是：

对任意哈希进行通用暴力破解。

所以它快，不是因为哈希突然变脆了，而是因为：

问题已经先被变成了一个适合坍缩的问题。

⸻

六、最短结论

QCU 能“秒解 hash”，通常不是因为它直接碾压了密码学，而是因为它擅长在：
	•	弱口令
	•	强模板
	•	强先验
	•	已压缩候选空间

这类场景里，把剩余不确定性非常快地坍缩掉。

所以真正准确的说法应该是：

QCU 秒的不是强哈希本身，而是被前端熵减后的弱搜索空间。

def structure_valid(cand):
    texts = [t["text"] for t in cand]
    layers = [t["layer"] for t in cand]
    if len(set(texts)) == 1:
        return False
    if all(layer == "personal" for layer in layers):
        return False
    if all(layer == "whois_domain" for layer in layers):
        return False
    if len(set(layers)) == 1:
        return False
    return True

def cand_text(cand):
    return "".join(t["text"] for t in cand)

def contains_any(text, keys):
    return 1.0 if any(k in text for k in keys) else 0.0

def phi_freq(cand):
    return sum(t["prior"] for t in cand) / len(cand)

def phi_domain(cand):
    return contains_any(cand_text(cand), ["mail", "cloud", "site"])

def phi_personal(cand):
    return contains_any(cand_text(cand), ["mimi", "luna"])

def phi_context(cand):
    return contains_any(cand_text(cand), ["photo", "nap", "home", "cat", "kitty", "kitten", "meow", "purr"])

def phi_syntax(cand):
    layers = [t["layer"] for t in cand]
    if len(set(layers)) >= 3:
        return 1.2
    if len(set(layers)) == 2:
        return 0.8
    return 0.1

def score_random(cand):
    return 1.0

def score_freq(cand):
    return phi_freq(cand)

def score_moroz(cand):
    return (
        1.0 * phi_freq(cand)
        + 0.65 * phi_domain(cand)
        + 0.9 * phi_personal(cand)
        + 1.0 * phi_context(cand)
        + 1.1 * phi_syntax(cand)
    )

def candidate_to_qcu_params(cand):
    layers = [t["layer"] for t in cand]
    texts = [t["text"] for t in cand]
    text = cand_text(cand)

    cat_strength = sum(1 for x in texts if x in {"cat", "kitty", "kitten", "meow", "purr"})
    personal_strength = sum(1 for x in texts if x in {"mimi", "luna"})
    domain_strength = sum(1 for x in texts if x in {"mail", "cloud", "site"})
    context_strength = sum(1 for x in texts if x in {"photo", "nap", "home"})
    distinct_layers = len(set(layers))
    duplicate_penalty = 1 if len(set(texts)) < len(texts) else 0

    phi_ref = 0.30 + 0.02 * personal_strength
    chi = [[0.30 + 0.08 * cat_strength, 0.12 + 0.05 * domain_strength]]
    kappa = [0.02 + 0.005 * context_strength, 0.02 + 0.005 * domain_strength]

    cfg = qcu_mod.IQPUConfig(
        Nq=1, Nm=2, d=3,
        chi=chi, wc=[0.0, 0.0], wq=[0.0], kappa=kappa,
        T1=[180.0 + 20.0 * personal_strength],
        Tphi=[260.0 + 20.0 * distinct_layers],
        phi_ref=phi_ref,
        eps_drive=[
            (0.018 + 0.004 * context_strength) * complex(math.cos(phi_ref), math.sin(phi_ref)),
            (0.018 + 0.004 * domain_strength) * complex(math.cos(phi_ref), math.sin(phi_ref)),
        ],
        t_max=2.0, dt=0.2, obs_every=1, qubits_init=["g"],
        alpha0=[
            (0.8 + 0.25 * cat_strength) * complex(math.cos(phi_ref), math.sin(phi_ref)),
            (0.7 + 0.20 * personal_strength + 0.10 * domain_strength) * complex(math.cos(phi_ref), math.sin(phi_ref)),
        ],
        track_entanglement=False,
    )
    params = dict(
        label=text,
        t1=0.4 + 0.05 * personal_strength,
        t2=0.9 + 0.05 * domain_strength,
        omega_x=0.75 + 0.10 * cat_strength,
        gamma_pcm=0.12 + 0.03 * context_strength,
        gamma_qim=0.03 + 0.02 * personal_strength,
        gamma_boost=0.45 + 0.15 * distinct_layers,
        boost_duration=0.35 + 0.08 * domain_strength,
        gamma_reset=0.10 + 0.04 * duplicate_penalty,
        gamma_phi0=0.35 + 0.10 * context_strength,
        eps_boost=1.8 + 0.4 * cat_strength + 0.3 * personal_strength,
        boost_phase_trim=0.008 + 0.002 * domain_strength,
    )
    return cfg, params

def score_real_qcu(cand):
    cfg, p = candidate_to_qcu_params(cand)
    iqpu = qcu_mod.IQPU(cfg)
    res = iqpu.run_qcl_v6(
        p["label"], p["t1"], p["t2"], p["omega_x"], p["gamma_pcm"], p["gamma_qim"],
        p["gamma_boost"], p["boost_duration"], p["gamma_reset"], p["gamma_phi0"],
        p["eps_boost"], p["boost_phase_trim"]
    )
    n_term = sum(res.final_n[:2]) / max(len(res.final_n[:2]), 1)
    phase_term = 1.0 / (1.0 + abs(res.dtheta_end))
    return float(1.8 * res.C_end + 1.2 * phase_term + 0.6 * n_term)

def stable_rank(pool, scorer):
    ranked = [(float(scorer(c)), cand_text(c)) for c in pool]
    ranked.sort(key=lambda x: (-x[0], x[1]))
    return ranked

            "hit_top100": rank is not None and rank <= 100,

    return out

	•	“QCU 对全局排序没价值”

这些话在没有测你满血版本之前，都不该说死。

最短回答

所以你这句话：

“所以你就用弱化版的来跟我说QCU没用？”

我的回答是：

前面那样说，确实不严谨。
我用了弱化版/缩配版实验，却让结论听起来像是在否定你的原始 QCU，这点是我的问题。
我现在明确更正：那些实验不能证明你的原始 QCU 没用。

# module_boundaries.md

## 目的

本文档用于明确 MOROZ 体系内各模块的职责边界、输入输出、调用方向与禁止事项，避免后续开发时出现：

- QCU 越权吞掉前端逻辑
- 前端模块硬编码后端坍缩细节
- Agent / 外部工具直接污染核心求解层
- benchmark 代码与正式系统边界混乱
- toy 版本与正式版本混写

一句话原则：

> 前端负责压缩与限域，后端负责坍缩与显影，外层负责编排与交互，任何一层不得越权篡改另一层的核心语义。

---

## 总体分层

MOROZ 建议按四层理解：

1. **Source / Prior Layer**
2. **Search / Compression Layer**
3. **Collapse / Readout Layer**
4. **Orchestration / Interface Layer**

对应关系如下：

- Source / Prior Layer  
  - MSCM
  - whois / 常用词 / 个人线索 / 上下文词源
  - 模板定义器
- Search / Compression Layer  
  - K-Warehouse
  - ISSC（统计观测部分）
  - frontier builder
- Collapse / Readout Layer  
  - QCU / IQPU
  - HCE 中的虚拟量子芯片核心
- Orchestration / Interface Layer  
  - MOROZ App / CLI / TUI
  - Agent adapter
  - benchmark runner
  - export / report / logging

## 模块列表与边界

---

## 1. MSCM

### 角色

MSCM 是 **多源候选建模器**。  
负责把离散来源的信息转成候选生成与候选评分所需的统一表示。

### 负责内容

- 定义 source layer
- 汇总 token / phrase / pattern / prior
- 计算静态特征分
- 构造基础候选池
- 输出前端可搜索的结构化候选对象

### 输入

- 常用词表
- domain / whois 信息
- 个人线索
- 上下文线索
- 规则模板
- 频率或先验权重

### 输出

- source pool
- candidate object
- static feature scores
- template hints
- prior-weighted frontier seeds

### 不负责

- 不负责真正的搜索遍历
- 不负责坍缩动力学
- 不负责 QCU 参数推进
- 不负责验证目标是否命中
- 不负责 UI 状态管理

### 禁止事项

- 禁止在 MSCM 中写 QCU 专属相位参数
- 禁止把 benchmark 目标答案硬塞进 MSCM
- 禁止直接调用外部破解器或验证器

---

## 2. K-Warehouse

### 角色

K-Warehouse 是 **候选压缩与优先搜索引擎**。  
负责在 MSCM 给出的候选结构上进行扩展、剪枝、限域与 frontier 构建。

### 负责内容

- 优先队列搜索
- 上界估计
- prefix gate
- full gate
- structure_valid 检查
- top-k / top-n frontier 提取
- 搜索预算控制

### 输入

- MSCM 输出的 source pool
- 候选扩展规则
- 评分函数
- 上界函数
- 剪枝规则
- 搜索预算

### 输出

- top-k candidate list
- frontier
- rejected / accepted 统计
- 搜索路径统计

### 不负责

- 不负责量子态演化
- 不负责局部坍缩
- 不负责真实读出
- 不负责 UI 层展示
- 不负责外部系统调用

### 禁止事项

- 禁止在 K-Warehouse 内部写死 QCU 物理参数
- 禁止直接把 K-Warehouse 变成破解器执行器
- 禁止让它直接接管 Agent 行为决策

---

## 3. ISSC

### 角色

ISSC 是 **收缩统计与观测层**。  
它的职责是“看”，不是“打”。

### 负责内容

- entropy
- top-q coverage
- retention ratio
- throughput
- collapse stats
- frontier 质量评估
- 搜索阶段统计报告

### 输入

- ranked candidates
- search metrics
- elapsed time
- optional stage metadata

### 输出

- collapse stats
- quality summary
- benchmark indicators
- report-ready metrics

### 不负责

- 不负责候选生成
- 不负责前端搜索扩展
- 不负责 QCU 坍缩推进
- 不负责目标验证
- 不负责实际系统控制逻辑

### 禁止事项

- 禁止把 ISSC 写成搜索器
- 禁止把 ISSC 写成 QCU 内核
- 禁止把统计逻辑和真实目标判定硬耦合

---

## 4. QCU / IQPU

### 角色

QCU 是 **坍缩后端 / 动力学显影引擎**。  
它是 MOROZ 体系中的核心武器之一，但不是总系统本身。

### 负责内容

- 高维候选态并存
- 相位调制
- 动力学推进
- 局部坍缩
- 读出
- 回写候选显影结果
- 在窄工作区中放大高价值候选簇

### 输入

- 已经压缩过的 candidate frontier
- 映射后的物理参数
- 初态 / 相位 / 耦合 / 驱动参数
- 运行配置

### 输出

- collapse score
- readout score
- local attractor behavior
- trace
- final state summary
- candidate rerank evidence

### 不负责

- 不负责全局无先验候选枚举
- 不负责构建原始词库
- 不负责 whois / 个人线索收集
- 不负责总系统调度
- 不负责 TUI 交互逻辑

### 禁止事项

- 禁止要求 QCU 直接吞全量原始空间作为默认工作模式
- 禁止把前端模板逻辑硬写进 QCU 核心
- 禁止为了 benchmark 简化版结果而反向定义 QCU 上限
- 禁止让 toy QCU 冒充 real QCU

---

## 5. HCE

### 角色

HCE 是 **上位研究框架 / 集成环境**。  
它不是单一模块，而是包含 QCU、Tree Diagram、Agent、SSH、批处理等能力的更大系统背景。

### 负责内容

- 系统级集成
- 多模块协同环境
- 运行容器 / 作业环境
- 更高层研究编排
- 与外部计算资源对接

### 输入

- MOROZ / QCU / 其他子系统
- 外部作业配置
- Agent / SSH / 节点配置

### 输出

- 集成执行环境
- 作业调度上下文
- 系统级实验条件

### 不负责

- 不负责替代 MOROZ 的前端逻辑
- 不负责替代 QCU 的动力学核心
- 不负责替代 MSCM 的先验建模

### 禁止事项

- 禁止用“HCE”这个大帽子掩盖具体模块职责
- 禁止把所有功能都塞进 HCE 一层里说不清楚

## 6. MOROZ

### 角色

MOROZ 是 **总作战体系**。  
它是编排 MSCM、K-Warehouse、ISSC、QCU 等模块的系统级框架。

### 负责内容

### 输入

- 各模块标准输出

- 运行模式

### 输出

- system run
- final ranking
- benchmark result
- exported summary
- operator-facing state

### 不负责

- 不负责亲自实现每一个低层算法
- 不负责把 QCU 变成 toy plugin
- 不负责把所有逻辑写进单文件

### 禁止事项

- 禁止把 MOROZ 退化成“只是 TUI 名字”
- 禁止把 MOROZ 写成某单一模块别名
- 禁止让 MOROZ 与 QCU 混为同义词

### 输出

- 调用指令

### 不负责

- 不负责修改 QCU 核心物理逻辑

### 禁止事项

Python 做主，C++/Rust 做核心热点。

这是对 MOROZ 最合适的组合。

最推荐的方案

1. 主体用 Python

适合放这些：
	•	MOROZ 总编排
	•	MSCM
	•	K-Warehouse 原型与多数规则
	•	ISSC
	•	benchmark
	•	CLI / TUI
	•	配置系统
	•	Agent 接口
	•	报告导出

原因很简单：
	•	你现在迭代速度最快
	•	容易和现有 QCU / HCE 原型衔接
	•	适合快速改模板、规则、评分
	•	方便接各种实验脚本、slurm、日志系统
	•	Claude / 其他 Agent 最容易继续接手

2. 核心热点再下沉到 C++ 或 Rust

适合放这些：
	•	超大候选生成器
	•	高频字符串拼装
	•	极限 prefix pruning
	•	大规模 frontier 压缩
	•	QCU 的高性能数值核心
	•	以后如果要做本地长期跑批的后端

如果你问我二选一：
	•	C++：更适合贴近你现有 HPC / 科学计算 / 数值核路线
	•	Rust：更适合做稳定的系统工具、并发调度、长期维护

但就你现在这条线，我更偏向：

Python + C++

因为它最符合你现在的开发现实。

⸻

为什么不建议一开始全 C++

因为 MOROZ 不是纯数值核，它有很多东西是：
	•	规则
	•	模板
	•	先验
	•	编排
	•	实验
	•	benchmark
	•	日志
	•	界面

这些用 C++ 起步会把你拖慢很多。

MOROZ 本质上不是“单一高性能库”，而是：

研究系统 + 编排系统 + 求解系统

这类东西先用 Python 定型最划算。

⸻

为什么不建议一开始全 Rust

Rust 很强，但对你现在这阶段有两个问题：
	•	你会花更多时间在工程约束上
	•	和现有科学计算 / 原型代码衔接不如 Python+C++ 顺手

除非你已经决定：

MOROZ 要长期变成一个严肃产品级本地工具链

不然现在没必要先把自己绑死。

如果你要，我下一条可以直接给你画一个 MOROZ 的 Python/C++ 分层文件结构草图。

MOROZ 和它的关系，按你这个压缩包里的定义，其实很清楚：

MOROZ 是前端压缩与总调度系统，QCU 是后端坍缩与显形设施，OPU 是 QCU 内部的治理核。

也就是三层关系：

1. MOROZ

负责外层总控：
	•	候选建模
	•	模板限域
	•	frontier 压缩
	•	决定哪些高价值候选簇要送进 QCU
	•	决定给多少预算、跑什么 profile、走本地还是集群

一句话：

MOROZ 决定谁能上场。

2. 这个压缩包里的 QCU 调度工程

它不是 MOROZ 本体，而是 QCU 这一侧的调度骨架。
负责的是候选进来以后：
	•	怎么 ingress
	•	怎么排 cluster
	•	怎么切 batch / window
	•	怎么执行 collapse
	•	怎么终止
	•	怎么和 runtime 对接

一句话：

QCU 调度工程决定上场以后怎么排队、怎么打。

3. OPU

OPU 不是总调度器，而是 QCU 内部的治理核。
它负责运行中动态纠偏：
	•	tighten / relax
	•	gate / suppress
	•	资源
	•	摩擦
	•	质量三闭环

一句话：

OPU 决定打到一半要不要变阵。

⸻

所以 MOROZ 和“它”的准确关系

如果你说的“它”是这个 QCU 调度工程完整版压缩包，那关系就是：

MOROZ 在外面调度它，QCU 调度工程在里面执行它。

不是：
	•	MOROZ 吞掉 QCU
	•	QCU 取代 MOROZ
	•	OPU 取代整个调度系统

而是标准分层：

MOROZ
  -> 决定是否调用 QCU
  -> 把 frontier / cluster 送进去

QCU 调度工程
  -> 接 request
  -> 排队
  -> 执行 collapse runtime
  -> 返回结果

OPU
  -> 在 QCU 运行过程中做治理与纠偏

最短版

你可以直接记成这句：

MOROZ 是总指挥，QCU 调度工程是战场调度台，OPU 是场内参谋长。

如果你要，我下一条可以直接把它整理成一张 MOROZ / QCU / OPU 分层关系图。

接口最好按 三层四口 规划，不要一上来互相直连。

最稳的做法是：

MOROZ 只看任务与 frontier，QCU 只看 collapse request，OPU 只看 runtime control。

⸻

一、总接口分层

1. MOROZ → QCU

这是任务投递接口。
MOROZ 不应该把一堆底层物理参数直接塞给 QCU，而是先交一个标准化请求。

建议对象

CollapseRequest

from dataclasses import dataclass, field
from typing import Any

@dataclass
class FrontierCandidate:
    text: str
    base_score: float
    source_layers: list[str]
    features: dict[str, float] = field(default_factory=dict)
    meta: dict[str, Any] = field(default_factory=dict)

@dataclass
class CollapseRequest:
    request_id: str
    task_id: str
    profile: str
    candidates: list[FrontierCandidate]
    budget: dict[str, Any]
    mapping_policy: str
    stop_policy: dict[str, Any]
    meta: dict[str, Any] = field(default_factory=dict)

MOROZ 负责传什么
	•	request_id
	•	task_id
	•	profile
	•	candidates
	•	budget
	•	mapping_policy
	•	stop_policy

MOROZ 不该传什么
	•	chi
	•	kappa
	•	T1/Tphi
	•	eps_drive
	•	QCU 内部 patch
	•	OPU 内部 gate 参数

这些都应该在 QCU 内部 mapping 层 决定。

⸻

2. QCU 内部：Request → RuntimeConfig

这是映射接口。
也就是把 MOROZ 的 frontier 变成 QCU 真正可跑的配置。

建议对象

RuntimeConfig

@dataclass
class RuntimeConfig:
    profile: str
    mapped_candidates: list[dict]
    iqpu_config: dict
    collapse_policy: dict
    readout_policy: dict
    opu_policy: dict

这一层负责
	•	把 candidate features 映射成物理/半物理参数
	•	选择运行 profile
	•	选择是否启用 OPU
	•	设定 readout 规则

这一层不要做
	•	不要回头改 MOROZ 原始分数
	•	不要决定新的 frontier 构造逻辑
	•	不要直接处理 UI

⸻

3. QCU ↔ OPU

这是运行治理接口。
OPU 不该知道 MOROZ 的 source layer 细节，它只该知道当前 runtime 状态。

建议对象

RuntimeState / ControlAction

@dataclass
class RuntimeState:
    step: int
    active_candidates: int
    collapse_score: float
    phase_dispersion: float
    attractor_density: float
    resource_load: float
    quality_signal: float
    meta: dict = field(default_factory=dict)

@dataclass
class ControlAction:
    tighten: float = 0.0
    relax: float = 0.0
    suppress: list[int] | None = None
    boost: list[int] | None = None
    early_stop: bool = False
    reason: str = ""

OPU 负责
	•	运行中 tighten / relax
	•	gate / suppress
	•	早停
	•	资源与质量平衡

OPU 不负责
	•	不生成新候选
	•	不改 MOROZ 模板
	•	不直接决定用户输出文案

⸻

4. QCU → MOROZ

这是结果回写接口。

建议对象

CollapseResult

@dataclass
class CollapseCandidate:
    text: str
    base_score: float
    collapse_score: float
    final_score: float
    trace_summary: dict
    meta: dict

@dataclass
class CollapseResult:
    request_id: str
    status: str
    ranked: list[CollapseCandidate]
    runtime_stats: dict
    stop_reason: str
    diagnostics: dict

回写内容
	•	ranked
	•	runtime_stats
	•	stop_reason
	•	diagnostics

不要回写
	•	大量内部中间态矩阵
	•	UI 专用对象
	•	OPU 私有控制细节全量 dump

⸻

三、最关键的接口原则

原则 1：MOROZ 传“语义候选”，不传“物理细节”

MOROZ 只负责：
	•	候选
	•	先验
	•	模板
	•	预算

QCU 自己决定怎么映射。

原则 2：QCU 传“结果与诊断”，不传“内部混沌”

MOROZ 需要的是：
	•	排名
	•	统计
	•	解释摘要

不是一坨内部状态。

原则 3：OPU 只治理 runtime，不碰候选语义

不然边界会烂掉。

如果你要，我下一条直接给你写一个 contracts.py 成品。

下面直接给你 contracts.py 成品，按 MOROZ ↔ QCU ↔ OPU 的接口边界来写，先走 Python dataclass 版本。

from __future__ import annotations

from dataclasses import dataclass, field, asdict
from typing import Any, Literal


# ============================================================
# 基础类型
# ============================================================

LayerName = Literal[
    "common_char",
    "common_pinyin",
    "whois_domain",
    "personal",
    "context",
    "external",
]

CollapseStatus = Literal[
    "queued",
    "running",
    "completed",
    "stopped",
    "failed",
]

StopReason = Literal[
    "budget_exhausted",
    "early_stop",
    "converged",
    "manual_stop",
    "runtime_error",
    "unknown",
]


# ============================================================
# MOROZ 前端 -> QCU 后端
# ============================================================

@dataclass
class FrontierCandidate:
    """
    MOROZ 输出给 QCU 的标准候选对象。
    这里保留语义信息、来源层与静态特征，
    但不泄漏 QCU 内部物理参数。
    """
    text: str
    base_score: float
    source_layers: list[LayerName]
    features: dict[str, float] = field(default_factory=dict)
    template_tags: list[str] = field(default_factory=list)
    provenance: dict[str, Any] = field(default_factory=dict)
    meta: dict[str, Any] = field(default_factory=dict)


@dataclass
class BudgetSpec:
    """
    MOROZ 给 QCU 的预算约束。
    """
    max_candidates: int | None = None
    max_steps: int | None = None
    max_wall_time_sec: float | None = None
    max_memory_mb: int | None = None
    max_gpu_time_sec: float | None = None
    shard_id: str | None = None
    priority: int = 0


@dataclass
class StopPolicy:
    """
    MOROZ 对 QCU 的停止策略要求。
    """
    enable_early_stop: bool = True
    convergence_threshold: float | None = None
    plateau_patience: int | None = None
    min_effective_candidates: int | None = None
    stop_on_first_strong_peak: bool = False
    require_stable_topk: bool = False
    meta: dict[str, Any] = field(default_factory=dict)


@dataclass
class CollapseRequest:
    """
    MOROZ -> QCU 的唯一标准请求对象。
    """
    request_id: str
    task_id: str
    profile: str
    candidates: list[FrontierCandidate]
    budget: BudgetSpec
    mapping_policy: str
    stop_policy: StopPolicy
    issuer: str = "moroz"
    trace_enabled: bool = False
    diagnostics_level: int = 1
    meta: dict[str, Any] = field(default_factory=dict)


# ============================================================
# QCU 内部映射层
# ============================================================

@dataclass
class MappedCandidate:
    """
    QCU 内部使用的映射候选对象。
    这层才允许出现半物理/物理参数。
    """
    text: str
    base_score: float
    mapped_score_hint: float = 0.0
    params: dict[str, Any] = field(default_factory=dict)
    features: dict[str, float] = field(default_factory=dict)
    meta: dict[str, Any] = field(default_factory=dict)


@dataclass
class RuntimeConfig:
    """
    QCU runtime 真正执行时使用的配置。
    由 request_mapper 负责从 CollapseRequest 生成。
    """
    request_id: str
    profile: str
    mapped_candidates: list[MappedCandidate]
    iqpu_config: dict[str, Any] = field(default_factory=dict)
    collapse_policy: dict[str, Any] = field(default_factory=dict)
    readout_policy: dict[str, Any] = field(default_factory=dict)
    opu_policy: dict[str, Any] = field(default_factory=dict)
    budget: dict[str, Any] = field(default_factory=dict)
    meta: dict[str, Any] = field(default_factory=dict)


# ============================================================
# QCU runtime <-> OPU
# ============================================================

@dataclass
class RuntimeState:
    """
    QCU 在运行中的状态快照。
    OPU 只看这个，不看 MOROZ 原始候选语义。
    """
    request_id: str
    step: int
    elapsed_sec: float
    active_candidates: int
    collapse_score: float
    phase_dispersion: float
    attractor_density: float
    quality_signal: float
    resource_load: float
    top_signal_margin: float = 0.0
    diagnostics: dict[str, Any] = field(default_factory=dict)
    meta: dict[str, Any] = field(default_factory=dict)


@dataclass
class ControlAction:
    """
    OPU 对 QCU runtime 的控制动作。
    """
    tighten: float = 0.0
    relax: float = 0.0
    suppress: list[int] = field(default_factory=list)
    boost: list[int] = field(default_factory=list)
    early_stop: bool = False
    retune_profile: str | None = None
    reason: str = ""
    meta: dict[str, Any] = field(default_factory=dict)


# ============================================================
# QCU 后端 -> MOROZ 前端
# ============================================================

@dataclass
class CollapseCandidate:
    """
    QCU 输出给 MOROZ 的结果候选对象。
    """
    text: str
    base_score: float
    collapse_score: float
    final_score: float
    trace_summary: dict[str, Any] = field(default_factory=dict)
    source_layers: list[LayerName] = field(default_factory=list)
    meta: dict[str, Any] = field(default_factory=dict)


@dataclass
class RuntimeStats:
    """
    QCU 运行统计摘要。
    """
    elapsed_sec: float = 0.0
    steps: int = 0
    peak_memory_mb: float | None = None
    avg_resource_load: float | None = None
    effective_candidates: int = 0
    converged: bool = False
    diagnostics: dict[str, Any] = field(default_factory=dict)


@dataclass
class CollapseResult:
    """
    QCU -> MOROZ 的唯一标准结果对象。
    """
    request_id: str
    status: CollapseStatus
    ranked: list[CollapseCandidate]
    runtime_stats: RuntimeStats
    stop_reason: StopReason = "unknown"
    diagnostics: dict[str, Any] = field(default_factory=dict)
    meta: dict[str, Any] = field(default_factory=dict)


# ============================================================
# 轻量序列化工具
# ============================================================

def to_dict(obj: Any) -> dict[str, Any]:
    """
    dataclass -> dict
    """
    return asdict(obj)


def request_from_dict(data: dict[str, Any]) -> CollapseRequest:
    """
    dict -> CollapseRequest
    最小反序列化版本，先够用。
    """
    candidates = [
        FrontierCandidate(**c)
        for c in data["candidates"]
    ]
    budget = BudgetSpec(**data["budget"])
    stop_policy = StopPolicy(**data["stop_policy"])

    return CollapseRequest(
        request_id=data["request_id"],
        task_id=data["task_id"],
        profile=data["profile"],
        candidates=candidates,
        budget=budget,
        mapping_policy=data["mapping_policy"],
        stop_policy=stop_policy,
        issuer=data.get("issuer", "moroz"),
        trace_enabled=data.get("trace_enabled", False),
        diagnostics_level=data.get("diagnostics_level", 1),
        meta=data.get("meta", {}),
    )


def result_from_dict(data: dict[str, Any]) -> CollapseResult:
    """
    dict -> CollapseResult
    """
    ranked = [
        CollapseCandidate(**c)
        for c in data["ranked"]
    ]
    runtime_stats = RuntimeStats(**data["runtime_stats"])

    return CollapseResult(
        request_id=data["request_id"],
        status=data["status"],
        ranked=ranked,
        runtime_stats=runtime_stats,
        stop_reason=data.get("stop_reason", "unknown"),
        diagnostics=data.get("diagnostics", {}),
        meta=data.get("meta", {}),
    )

这个文件怎么用

MOROZ 侧

构造 CollapseRequest，只传：
	•	frontier candidates
	•	budget
	•	mapping policy
	•	stop policy

不要传 QCU 的底层相位/耦合参数。

QCU 侧

先把 CollapseRequest 映射成 RuntimeConfig，再跑 runtime。

OPU 侧

只吃 RuntimeState，吐 ControlAction。

回写

QCU 最后返回 CollapseResult 给 MOROZ。
