# MOROZ 神秘感品牌叙事计划书（合法公开版）

## 目的

本文档用于为 MOROZ 设计一套**合法、公开、可控**的品牌叙事与传播方案。  
目标不是制造违法用途或暗网交易，而是营造一种：

- 冷硬
- 稀缺
- 黑箱研究设施
- 机密计划代号
- 都市传说感

的品牌气质。

一句话定位：

> 把 MOROZ 传播成“像传说一样的研究系统”，而不是“可疑的地下交易商品”。

---

## 核心原则

1. **不触碰违法市场、不使用暗网售卖叙事**
2. **不宣传破解、攻击、绕过等可操作用途**
3. **把重点放在系统架构、研究框架、控制台气质、黑箱设施感**
4. **传播“稀缺 access 感”，而不是传播违法 access**
5. **所有材料都可在公开平台发布**

---

> MOROZ 不是一个普通工具，而是一套把模糊问题压成可显影工作区的黑箱研究系统。

## 都市传说感的合法制造方式

### 可做的
- 使用“window / access / operator / handoff”这类系统语言
- 用 run summary 残片营造黑箱感
- 用模块边界与接口契约增强真实感
- 用“不是人人都配乱开后端”这种系统语气营造稀缺感
- 用限量 demo、限时公开文档制造窗口感

### 不可做的
- 暗示非法渠道
- 暗示购买地下服务
- 编写引导违法接触方式
- 兜售真实破解能力
- 伪造成功案例诱导交易

## 风险控制

### 风险 1：被误读为非法工具宣传
应对：
- 一切公开页面都强调研究框架、系统设计、benchmark、边界
- 不展示真实攻击叙事
- 不写面向实际破解的操作内容

### 风险 2：过度神话，显得像吹牛
应对：
- 白皮书先行
- benchmark 先行
- 模块边界先行
- 少说“神器”，多说“系统”

### 风险 3：品牌太硬，受众看不懂
应对：
- 配一页极简“这是什么”
- 把术语翻成用户能懂的层级语言
- 给出示意图而不是全靠黑话
