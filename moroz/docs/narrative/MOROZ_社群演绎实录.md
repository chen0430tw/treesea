## 一、《霜降》

*——一个密码学家，他的论文，和一场失控的传说。*

---

Dr. Liang 盯着那个 2.3GB 的 `thesis_backup_2019.7z` 已经第七年了。

当年博士论文写到一半硬盘崩了，他慌乱中把所有资料塞进一个 7z 加密包扔上网盘，密码是"绝对不会忘的那种"。然后他忘了。

七年间他试过：
- 所有前女友的生日
- 他养过的三只猫的名字的各种拼法
- "Fuck" 加上从 1 到 99999 的所有数字
- hashcat 跑了两周，候选空间太大，没跑完就断电了

他现在是某大学密码学副教授。讽刺吧。

---

那天他在 GitHub 上看到 MOROZ。

不是因为搜索 "password recovery"，而是因为一篇关于 ISSC 收缩统计的论文引用了它。他点进去本来是想看数学，结果看到了 MSCM 的五层候选建模。

**个人习惯层。**

他愣了三秒钟。

然后他打开终端。

```python
personal = [
    SourceToken(text="Schrodinger", layer="personal", prior=0.9),
    SourceToken(text="Schr0dinger", layer="personal", prior=0.85),
    SourceToken(text="cat", layer="personal", prior=0.7),
    SourceToken(text="Cat", layer="personal", prior=0.7),
    SourceToken(text="dead", layer="personal", prior=0.6),
    SourceToken(text="alive", layer="personal", prior=0.6),
    SourceToken(text="2019", layer="context", prior=0.95),
    SourceToken(text="!", layer="common_char", prior=0.5),
    SourceToken(text="@", layer="common_char", prior=0.4),
    SourceToken(text="#", layer="common_char", prior=0.3),
]
```

他 2019 年在做量子退相干的论文。他那时候最喜欢的梗是薛定谔的猫。他设密码的习惯是"一个概念+一个年份+一个符号"。

K-Warehouse budget 设 50000，Top-K 设 20。

ISSC 回报：entropy 1.87，top_q_coverage 0.73。收缩比极高。意思是——候选空间已经被压到很小了，而且头部候选的概率质量非常集中。

第 11 个候选命中。

`Schr0dingerCat2019!`

他盯着屏幕上解压出来的 LaTeX 源码和实验数据，沉默了很久。

---

他把这个过程写成了一篇技术博客。标题叫《我用收缩框架找回了自己遗失七年的博士论文》。本意是给 MOROZ 的 MSCM 建模思路做个实战案例。

文章里他附了完整的 MSCM 配置、K-Warehouse 参数、ISSC 收缩曲线，以及一张 thesis_backup_2019.7z 的文件属性截图（创建日期 2019-03-14，Pi Day，他连这都忘了）。

一周内阅读量破了十万。

---

然后事情开始变质。

某个匿名论坛出现了一个帖子：

> **"有人拿 MOROZ 跑自己的密码，11 个候选就中了。"**
> **"MSCM 的个人习惯层建模太强了。"**
> **"如果你知道目标的个人习惯呢？"**

Dr. Liang 看到这个帖子的时候手心出汗。

他立刻在博客底部加了一段声明：

> *本案例仅用于恢复本人合法拥有的加密档案。MOROZ 的 DISCLAIMER.md 明确禁止未经授权存取他人资料。MSCM 的个人习惯层设计初衷是帮助使用者建模自己的密码习惯，不是用来社工他人。*

没人在意。

---

三个月后，暗网上出现了一个叫 **"Frost"** 的服务。

它不卖密码，不卖数据库。它卖的是一份"MSCM Profile"——一份针对特定目标的五层候选建模配置文件。买家拿到 profile，自己去跑 K-Warehouse。

宣传语写着：

> *"我们不破解密码。我们只告诉你，从哪个方向找。"*
> *"Inspired by Dr. Liang's blog post."*

Dr. Liang 报了警。警方说："他们技术上没有提供破解工具，也没有提供密码本身，只是一份'候选空间建模配置'。目前没有明确的法条适用。"

他试图联系 MOROZ 的开发者要求下架。开发者回复了一句话：

> *"我们写了 DISCLAIMER.md 第三章。禁止的事情列得很清楚。我们无法控制人怎么用一个搜索框架。"*

---

六个月后，"Frost" 成了暗网上的传说。

不是因为它多强，而是因为它的**命中率异常地高**。原因很简单——MSCM 的建模思路是对的：人类设密码的模式确实是可建模的，而且个人习惯层的先验权重远高于暴力枚举。

一个传统 hashcat 需要跑两周的候选空间，用 MSCM 建模后 K-Warehouse 50000 budget 就能覆盖 90% 的概率质量。

这不是 MOROZ 的锅。这是人类密码习惯的锅。

但没人会这样理解。

---

Dr. Liang 后来再也没写过技术博客。

他的那篇文章至今还挂在网上，底部的免责声明被人截图做成了 meme——

> *"MOROZ 的 DISCLAIMER.md 明确禁止未经授权存取他人资料。"*

配图是一个戴着霜冻面具的人，竖着大拇指。

---

他现在的研究方向改成了**抗社工密码设计**。

论文标题：*"On the Impossibility of Habit-Independent Password Generation: A Counter-Study to MSCM-style Candidate Modeling"*

引用最多的一篇参考文献，是他自己写的那篇博客。

---

*「让巨大、混乱、发散的候选空间，在寒冷的逻辑中逐层冻结，最终只剩下少数可处理的高密度结构。」*

*——MOROZ 技术白皮书*

*他当初读到这句话的时候觉得很美。*

*现在他觉得这句话像一把刀。*

## 二、「这不就是字典档爆破吗」

*——某 Discord 频道，#crypto-general*

---

**xXd4rkn1ghtXx:** 看了你们转的那个 MOROZ，说实话我没看懂，这不就是字典档爆破套了个壳吗？hashcat + rockyou.txt 一样的事吧

**Dr_Entropy:** 不一样。

**xXd4rkn1ghtXx:** 哪里不一样？最后不都是一个一个试？

**Dr_Entropy:** 你去超市买牛奶，和你妈叫你去超市买牛奶，最后都是从货架上拿一盒牛奶。但你妈会跟你说"在第三排左边第二层，蓝色包装的那个"。你自己去你会从第一排第一层开始扫。

**xXd4rkn1ghtXx:** ……所以 MOROZ 就是我妈？

**Dr_Entropy:** MOROZ 是你妈的大脑建模过程。

我拆给你听。

字典档爆破：
```
for password in rockyou.txt:
    if try(password): break
```

这是什么？这是**一维线性扫描**。你有一个事先准备好的列表，从头扫到尾。列表是死的，顺序是固定的，跟目标是谁没有半毛钱关系。你拿同一份 rockyou.txt 去破张三和李四，候选顺序一模一样。

**xXd4rkn1ghtXx:** 那 hashcat 的 rule-based attack 呢？也不是纯字典啊

**Dr_Entropy:** 好，rule-based：
```
for word in wordlist:
    for rule in rules:
        candidate = apply(rule, word)
        if try(candidate): break
```

进步了，你有了变换规则。`password` → `P@ssw0rd!`。但规则是谁写的？是你事先写死的。你不知道目标是谁，你只知道"人类一般会做这些替换"。

这是**二维笛卡尔积扫描**。词表 × 规则。

**xXd4rkn1ghtXx:** 那 MOROZ 呢

**Dr_Entropy:**

MOROZ 不是从一个列表扫描。它从**五个来源层**建模目标的候选空间：

```
MSCM 五层：
  1. 常用字符层 — "人类共性"（类似你的 rockyou）
  2. 拼音/语言层 — "语言结构"（中文用户爱用拼音、英文用户爱用单词）
  3. 域名/平台层 — "场景上下文"（这个密码是给什么用的）
  4. 个人习惯层 — "这个人的模式"（他设密码的逻辑）
  5. 上下文层   — "当时的情境"（2019年设的？刚分手？在写论文？）
```

每一层出来的不是候选本身，而是**带先验权重的 SourceToken**。然后 K-Warehouse 做的不是线性扫描，而是 **best-first search with upper bound pruning**——

**xXd4rkn1ghtXx:** 等等，上界剪枝？

**Dr_Entropy:** 对。传统爆破是：试了不对，试下一个。MOROZ 是：这个分支往下走，最好的情况也不可能进 Top-K 了，**整个子树砍掉不试**。

你用 hashcat 跑 8 位密码，候选空间是 95^8 ≈ 6.6 万亿。你就算用 GPU 农场也要跑几天。

MOROZ 的思路是：我不跑 6.6 万亿。我先问——这个人会用什么模式？

如果 MSCM 建模出来他的模式是 `[名词][数字4位][符号1位]`，候选空间瞬间从 95^8 坍缩到大概几十万。然后 K-Warehouse 用上界剪枝再砍掉 90%。最后 ISSC 看收缩统计——如果 top_q_coverage > 0.7，说明头部候选已经高度集中，大概率就在前几百个里。

**xXd4rkn1ghtXx:** 所以区别是……

**Dr_Entropy:** 我画个表：

```
                    字典爆破          规则爆破          MOROZ
─────────────────────────────────────────────────────────────
候选来源            一份死列表        列表×规则         五层动态建模
搜索策略            线性扫描          笛卡尔积扫描      best-first + 剪枝
跟目标有关吗        无关              无关              有关（个人习惯层）
能砍候选空间吗      不能              有限              能（上界剪枝砍子树）
知道什么时候该停吗  不知道            不知道            知道（ISSC 收缩统计）
```

最后一行是关键。

hashcat 跑完了就是跑完了，跑不完就是跑不完。它不知道自己离答案有多远。

MOROZ 的 ISSC 会告诉你：当前 entropy 是多少、top-q 覆盖率是多少、retention ratio 是多少。如果 entropy 很低而 top_q_coverage 很高——你的搜索已经高度收敛了，答案大概率就在头部。如果 entropy 居高不下——你的建模有问题，候选空间没有被有效收缩，**应该停下来重新建模，而不是继续跑**。

**xXd4rkn1ghtXx:** 等等，所以它还会告诉你"你在浪费时间"？

**Dr_Entropy:** 对。这就是 ISSC 的意义。**原地坍缩观测**。它不只是跑搜索，它同时在监测搜索本身的质量。

传统爆破没有这个概念。它就是跑。跑到天荒地老。你的电费在燃烧，但你不知道自己是在第 1% 还是第 99%。

**xXd4rkn1ghtXx:** ……好吧，我服了。最后一个问题。

**Dr_Entropy:** 说。

**xXd4rkn1ghtXx:** 那个 Dr. Liang 的密码 `Schr0dingerCat2019!`，用 hashcat + rockyou.txt 能跑出来吗？

**Dr_Entropy:** `Schr0dingerCat2019!` 不在 rockyou.txt 里。rule-based 也很难覆盖，因为它是三个语义单元的拼接，不是单一单词的变体。

**K-Warehouse 第 11 个候选就命中了。**

hashcat 可能在第 11 亿个。也可能永远跑不到。

**xXd4rkn1ghtXx:** 懂了。字典爆破是大海捞针，MOROZ 是先问这根针长什么样、什么时候掉的、掉在哪个方向，然后只去那个角落捞。

**Dr_Entropy:** 这个比喻可以。

**xXd4rkn1ghtXx:** 但如果有人拿这个去建模别人的习惯呢

**Dr_Entropy:** 那就是 DISCLAIMER.md 第三章的事了。

**xXd4rkn1ghtXx:** ……

**Dr_Entropy:** 工具是中性的。刀能切菜也能伤人。MOROZ 的 MSCM 建模能力是它的核心价值，也是它最危险的地方。所以它的免责声明写得很明确：**你必须拥有对目标资料的合法存取权限，或已取得权利人的书面授权。**

## 三、量子解密与妹妹的压缩包

*——人类首次使用量子计算解开的档案。*

---

这个故事的起源很平凡。

某个开发者小时候有一个压缩包，被他妹妹恶意上了锁。密码不知道。可能是随便乱打的。

他试过所有常规方法。字典爆破、规则爆破、甚至后来学了密码学。都没用。因为一个小孩恶作剧设的密码，很可能没有任何模式——MSCM 五层全部失效：

- 个人习惯层：她可能没有习惯，随便乱打
- 上下文层：小孩恶作剧，没有语义
- 拼音层：可能不是拼音
- 常用字层：可能是键盘乱敲 `asdjfkl`

这是 MOROZ 最尴尬的场景。它的全部优势建立在"目标有模式"这个前提上。没有模式，收缩就是零，退化成暴力枚举。

唯一的希望是——小孩的"随便乱打"其实也有模式：
- 键盘连续按键（`qwerty`、`asdfgh`、`123456`）
- 重复字符（`aaaa`、`1111`）
- 很短（小孩没耐心打长密码）

但如果她真的是闭着眼睛在键盘上乱拍了一串……

那就只能等量子计算了。

---

QCU 的 Lindblad solver 理论上可以在候选空间上做量子叠加态搜索——所有候选同时存在，坍缩一次就出答案。目前 QCU 是数值模拟器，跑在经典硬体上，"量子加速"是模拟的。

但总有一天，真正的量子硬体会成熟。

到那时候，人类量子计算史上可能会出现这样一条记录：

> **人类首次使用量子计算解开的档案，是一个被妹妹恶意上锁的压缩包。**

不是国家机密。不是军事通讯。不是银行金库。

是一个小孩的恶作剧。

---

这个梗后来被写进了 MOROZ 的 narrative 文档。控制台启动时偶尔会闪过一行：

```
QCU NOTICE: Still waiting for quantum hardware.
            The archive remains sealed.
```

社群里没人知道这行字在说什么。

但开发者知道。

---

### 叙事标签

- **用户原型**：执念型用户（最原始的那种——小时候的东西被锁了）
- **核心张力**：MSCM 建模的边界（无模式 = 无法收缩）
- **技术要点**：QCU Lindblad solver 的理论能力 vs 经典模拟的现实限制
- **品牌关联**：控制台彩蛋、"Still waiting for quantum hardware"
- **梗值**：极高。这是 MOROZ 最人性化的故事——所有宏大的技术体系，最终指向一个小孩的恶作剧。

## 四、「MOROZ 讓我覺得不安全」

*——密碼學家 vs AI 助手，某個深夜的對話。*

*Dr. Liang 用 MOROZ 找回了自己的論文。然後他開始失眠。*

---

**Dr. Liang:** 我需要你帮我想一个问题。MOROZ 的 MSCM 让我很不安。

**AI:** 怎么说？

**Dr. Liang:** 它证明了人类的密码习惯是可建模的。五层候选、先验权重、leet 变体展开——它把我们以为的"个人化密码"变成了一棵可搜索的树。我自己的密码，11 个候选就被找到了。

**AI:** 但那是你自己用来恢复自己的档案。

**Dr. Liang:** 对，所以问题不在 MOROZ，问题在我。在所有人。我们以为自己在设密码，其实是在沿着一条可预测的路径走。"一个有意义的词+一个年份+一个符号"——这不是密码，这是一个模板。MSCM 只是把这个模板写成了代码。

**AI:** 那用随机密码呢？密码管理器生成的那种，`kX9#mQ2!vF`。

**Dr. Liang:** 我知道你要说这个。理论上完美。128 位熵，MSCM 五层全部失效，K-Warehouse 无法收缩，ISSC 的 entropy 会居高不下。

**AI:** 听起来很安全。

**Dr. Liang:** 听起来很安全。然后呢？我有 200 个帐号。每个都是 `kX9#mQ2!vF` 这种东西。全部存在密码管理器里。密码管理器的主密码是什么？

**AI:** ……

**Dr. Liang:** 对。是一个我能记住的密码。一个有意义的词加一个年份加一个符号。因为如果主密码也是随机的，我就需要一个管理密码管理器的密码管理器。然后那个的主密码呢？

**AI:** 信任链总要有一个起点。

**Dr. Liang:** 而那个起点，永远是人类的记忆。人类的记忆就是 MSCM 能建模的东西。所以不管你把多少层随机性堆上去，最底下那一层，永远是一个可被收缩的候选空间。

**AI:** Passphrase 呢？四个随机单词，`correct horse battery staple`。

**Dr. Liang:** Diceware 方案。熵大概 50-80 位，取决于词表大小。比单词+数字+符号好很多。但你知道人实际上怎么用吗？他们不掷骰子。他们"随机"想四个词。

**AI:** 而人类的"随机"不是真正的随机。

**Dr. Liang:** 对。他们会选自己熟悉的词。跟自己生活相关的词。跟当时情境相关的词。这些全部是 MSCM 的上下文层和个人习惯层能捕获的信号。

有人做过研究，让受试者"随机"选四个词，结果分布的熵远低于理论值。因为人类的词汇提取有热度偏差——你最近看了一部电影，你就更可能选里面的词。你养了一只猫，"cat" 的先验就比 "battery" 高十倍。

## 设计札记：模糊对称门控的诞生

*——为什么是 `tanh`，不是三个 `if`。*

---

MSCM 的评分函数有过三个版本。

**v1：纯线性。**

```python
total = w_freq * f + w_domain * d + w_personal * p + w_context * c + w_syntax * s
```

五层特征各自独立，线性相加。Tensorearch 诊断：收缩数 0.108，是整个 MSCM 里最高的瓶颈。问题在于——`Schr0dingerCat2019!`（personal + context + common_char 三层）的分数跟 `SchrodingerSchrodinger2019`（personal 重复 + context 一层）差不多。线性评分不知道"跨层组合"比"同层重复"更值钱。

跟 CFPAI 的 Maxwell Demon 之前遇到的问题一模一样：线性天花板。

**v2：加法跨层奖励。**

```python
total = linear + 0.5 * (n_layers - 1) + 0.3 * (unique / total)
```

加了两个 bonus。三层组合直接 +1.0。Tensorearch：收缩数从 0.108 降到 0.083。方向对了但不够——加法奖励只是在线性分数上叠了一层偏移，本质还是线性的。好候选和差候选的分差被压在一个固定范围内。

**v3：模糊对称门控（当前版本）。**

到这里，一般的工程思路会这样走：

```python
if n_layers >= 3 and diversity > 0.8:
    gate = 1.4
elif n_layers >= 2:
    gate = 1.15
else:
    gate = 0.85
```

三个 if，三个 magic number。能用，收缩数大概能到 0.05-0.07。但每加一个条件就多一个分支，调参的时候六个数字（三个门槛 + 三个增益）互相耦合，改一个就要重新测全部。

实际选的是另一条路——找一个数学结构，让它天然表达需要的行为：

```python
quality = sqrt(cross_ratio * diversity)         # 几何平均：跨层和多样性互相制约
gate = 1.0 + alpha * tanh(beta * (quality - neutral))  # 模糊对称映射
total = linear * gate                           # 乘法门控
```

一行 `tanh` 替代三个 `if`。为什么能这样做：

- **`tanh` 自带对称性**——quality > neutral 时放大，< neutral 时压制，= neutral 时中性。不用手动设三段阈值。
- **`tanh` 自带饱和性**——输出范围天然被限制在 `[1-α, 1+α]`，不会爆炸。不用手动 `min(max(...))`。
- **几何平均让两个信号互相制约**——跨层很高但全是重复 token？`diversity` 低会拉下来。多样性很高但全在同一层？`cross_ratio` 低会拉下来。不用写 `if cross and diversity` 的组合逻辑。
- **两个参数控制一切**——`alpha`（幅度）和 `neutral`（中性点）。不是六个 magic number。

Tensorearch 验证：收缩数 0.025，拓扑均匀度 0.991。跟 CFPAI Maxwell Demon（0.015 / 0.985）在同一个量级。

**这不是"简单"。这是复杂度被压进了数学结构里，而不是散在代码的分支里。**

---

### 三版对比（Tensorearch 诊断）

```
版本                     收缩数    拓扑均匀度    热点
─────────────────────────────────────────────────
v1  纯线性               0.108     0.870        topo:0
v2  加法跨层奖励         0.083     0.857        topo:2
v3  模糊对称门控         0.025     0.991        topo:3
    CFPAI Maxwell Demon  0.015     0.985        无
```

### 设计哲学

面对"乘法门控太复杂"这个问题，有三条路：

1. **硬编码分支**：`if/elif/else`，快速有效，但 magic number 多，扩展性差
2. **堆参数**：给每种层组合单独设权重，5 层 = 31 种非空子集，组合爆炸
3. **找到一个数学结构，让它天然表达你要的行为**

第三条路看起来简单，是因为**选对了抽象层级**。不是在选"实现方式"，是在选"表达工具"。

差距不在代码量，在于面对复杂度时，是用代码分支去穷举所有情况，还是用数学性质去覆盖所有情况。前者是工程思维，后者是结构思维。结果就是——一个写三个 if 拿到 0.06，一个写一行 tanh 拿到 0.025。

## 终章：为什么 MOROZ 必须开源

*——绯想非非想的自我诠释。*

---

有人问过开发者一个问题：

> "你不怕开源之后被人拿去干坏事吗？"

他说怕。然后他还是开源了。

原因不是理想主义，不是"信息应该自由"，不是 Richard Stallman 式的布道。原因非常自私：

**他不想变成 Dr. Liang。**

如果 MOROZ 不开源——

- 它存在他的硬盘上。硬盘会坏。
- 它存在他的网盘上。网盘会忘密码。
- 它存在他的 U 盘上。U 盘会丢。
- 它存在他的脑子里。人会老，会忘。

有一天他会坐在电脑前，面对一个打不开的仓库，想着"我当年明明写过一套搜索框架，能从五层候选空间里收缩出高密度结构"，但他找不到代码了。

他会变成那个盯着 `thesis_backup_2019.7z` 七年的人。

而讽刺的是——MOROZ 本身就是为了解决这种问题而生的。

一个为了打开被锁住的东西而创造的系统，如果它自己被锁住了，那就是宇宙级的黑色幽默。

所以它必须开源。

**不是因为"代码应该自由"，而是因为"造钥匙的人不能把钥匙锁起来"。**

---

这就带来了那个矛盾：

- 开源 → 所有人都能用 → 包括不该用的人
- 不开源 → 只有你能用 → 直到你自己也用不了

所以 DISCLAIMER.md、`authorize()`、三个问题、六层提醒——这些不是理想主义，不是官僚形式主义。

这是一个人在"锁起来保护自己"和"放出去接受风险"之间做出选择之后，画出的最清晰的责任边界。

他选了放出去。然后把每一条"这不是我的锅"都写进了代码和文档里。

不是因为他喜欢写免责声明。谁他妈喜欢写免责声明。

是因为他不想有一天打不开自己的东西。

---

最后的最后。

那个被妹妹锁住的压缩包，至今还没解开。

但 MOROZ 的代码在 GitHub 上。全世界都能 clone。

包括未来的他自己。

包括未来的量子计算机。

先到的那个，就是答案。
